3.7 Hyperlinks en anchors

We maken gebruik van een <a> tag om links te leggen naar andere adressen (mail/web) of een inline page adres (anchor)

1.4 Het document object model

Dit is een functie binnen in je browser console die een hiërarchische structuur van een HTML pagina aantoont

- De li elementen zijn de children van het ul element

- Het ul element is de parent van de li elementen

- Het body element is de ancestor van het h1 element, het p element, het ul element en de li elementen

- p, h1, ul , li en body zijn descendants van het html element

- De li elementen zijn siblings

2: Selectors:

2.2 Element selector:

2.2.1 Enkelvoudige-selector

We selecteren h1 en passen hier dingen op toe

h1 {color: red; background-color: orange;

2.2.2 Meervoudige-selector

We selecteren hier h2 en li en passen hier dingen op toe

h2, li {font-size:80%; color:white;}

2.2.3 Descendent-selector

Alle em elementen zullen zwart zijn behalve de em elementen die als parent p hebben

(Wordt toegepast op een element dat afstamt van een voorgaand element)

p em {color:white;}

2.2.4 Child-selector

Deze stijl wordt alleen toegepast als het element direct afstamt van de parent

Hier gaat em rood kleuren als het direct afstamt van p, dus niet als er bijvoorbeeld een bold tag tussen staat

p > em {color:red;}

Adjencent sibling selector

Een h2 die onmiddelijk na een h1 komt in de dom tree van de pagina zal 5mm minder boven-marge hebben dan een gewone h2

h1 + h2 {margin-top:-5mm;}

2.3 Class en id selector

- Een class selector mag een onbepaald aantal keer voorkomen in een document

- Een id mag maar 1 keer voorkomen

Classes

Alle elementen die de class bevatten zullen de stijldeclaratie hebben

html:

<element class="class-naam">

css:

.class-naam {stijldeclaratie}

Alle p elementen die de klasse voorbeeld bevatten hebben de stijldeclaratie

<p class="voorbeeld"> Voorbeeld </p>

p.voorbeeld {stijldeclaratie}

ID's

Het element dat de id bevat heeft de stijldeclaratie

<element id="id-naam">

#id-naam {stijldeclaratie}

Het element p met id voorbeeld heeft de stijldeclaratie

<p id="voorbeeld"> Voorbeeld </p>

p#voorbeeld {stijldeclaratie}

2.4 Pseudo-element selectors

Dit kunnen we gebruiken als we niet op alles van een element iets willen toepassen maar bv op elke eerstel letter van een zin

element::pseudo-element{declaratie}

  • ::first-line = De eerste regel van een tekst apart opmaken
  • ::first-letter = Hetzelfde als firstline maar de eerste letter
  • ::before / ::after = Met deze twee elementen, kunnen we dingen toevoegen voor of na de tekst, bv een afbeelding
  • ::selection = Als de gebruiker in zijn browser iets selecteerd, dan bepalen we de opmaak hiervan

7.1 Normal flow

Elk element heeft een margin, een border, een padding en zijn inhoud en beschouwen we als een rechthoek

7.3 Visual formatting

Dit model bepaalt of de blokken naast of onder elkaar komen te staan

Alinea's komen standaard onder elkaar te staan, net zoals een lijst, maar afbeeldingen en links worden naast elkaar geplaatst (tot de regel vol is)

P of H1 worden dus als "block elementen" beschouwd => onder elkaar geplaatst

- Inline = elementen die naast elkaar staan

- Inline block = block elementen staan naast elkaar in een line

7.4 Positioneren

7.4.1 Zwevende elementen (float)

Met de float eigenschap wordt het element uit de normale flow van de pagina gehaald (!!verplicht om een width mee te geven!!)

Het gaat rond "zweven", aan de hand van de waarde:

float: left/right/none;

Pas op de elementen die volgen op het floating element weten niet dat dit rond zweeft (gaat buggen)

Als we float: clear; toepassen dan cancellen we de float op de volgende elementen

7.4.2 Relatief positioneren

position: relative;

De plaats van het element wordt bepaald ten opzichte van de gebruiklijke positie van het element

De voorziene ruimte blijft dus leeg maar het element verplaatst zich wel

7.4.3 Absoluut positioneren

position: absolute;

We kunnen met coordinaten bepalen waar het staat ten opzichte van (voor-ouder) elementen dat een eigenschap hebben dat niet static is

Als er niet zo een element aanwezig is dan wordt het berekend ten opzichte van de box van het body

Geven we geen coordinaten mee dan is het automatisch top:0; en left:0;

7.8 CSS grid

Indeling

We delen adhv de pagia/container op in een raster

Hiermee kunnen we makkelijk de pagina indelen

.container{display: grid;}

- grid-template-columns = bepalen van de kolommen

- grid-template-rows = bepalen van de rijen

- grid-template-areas = de vakken een naam geven om te gebruiken

.container{display: grid; grid-template-colums: 1fr 1fr 1fr; grid-template-rows: 1fr 1fr 1fr; gap: .5rem}

Een grid 3x3 met gaps van .5rem spatie tussen de cellen

We bakenen alles af met lijnen en bepalen hiermee ook de positie

.item{grid-column-start: 1; grid-column-end: 3; grid-row-start: 1; grid-row-end: 3;}

De eerste 2 vakjes van de 1ste rij en de 2de rij zijn nu .item

7.9 FlexBox

Algemeen

Flexbox is een algemeen eendimensioneel lay-out systeem (werkt dus altijd maar in 1 richting)

De elementen worden op de main axis en de cross accis geplaatst, dit speelt een grote rol want hiermee kunnen we ze positioneren

7.9.1 Flex container: display

We leggen vast dat de container die de elementen bevat met flex werken

.container {display: flex;}

7.9.2 Flex container: flex-direction

De richting waarin de flex elementen worden geplaatst => flex-direction

.container {flex-direction: row | row-reverse | column | column-reverse;}

7.9.3 Flex container: flex-wrap

Hiermee bepalen we de omloop van de elementen (standaard is er geen omloop)

.container {flex-wrap: nowrap | wrap | wrap-reverse;}

7.9.5 Flex container: justify-content

.container {justify-content: flex-start | flex-end | center | space-between | space around;}

7.9.6 Flex container: align-items

.container {align-items: flex-start | flex-end | center | baseline | stretch;}

7.9.7 Flex container: align-content

Als je container over meerdere rijen bestaat

.container {justify-content: flex-start | flex-end | center | space-between | space around | stretch;}
